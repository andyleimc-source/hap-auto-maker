HAP Application Lifecycle Automation

把“描述需求”变成“应用 + 数据 + 演示视频”

HAP Auto 是一个围绕明道云 / HAP 的自动化系统。它不是单点脚本，而是一套将业务需求、应用结构规划、界面配置、示例数据和浏览器演示录制串成闭环的流水线。

一句话定义
从“我想做一个什么应用”开始，自动生成应用骨架、工作表、字段布局、视图、筛选、示例数据，并产出一段可展示给外界的操作视频。
3 段 核心自动化闭环：需求、构建、录制
2 类 输出结果：业务结果和技术日志
1 条 主入口：run_app_to_video.py
全链路 从应用策划一直走到视频演示成片
项目适合什么场景
  • 快速制作行业应用 Demo
  • 自动搭建内部试运行应用
  • 给客户、老板、团队展示“从需求到结果”的全流程能力
Project Value

这个项目解决的不是单一步骤，而是整条链路

外界通常看到的只是“生成一个应用”或者“录一个视频”。HAP Auto 处理的是这两者之间的大量中间工作：需求结构化、方案规划、配置执行、视图生成、造数、浏览器操作和结果归档。

把业务描述转成系统动作

用户先说“我想要什么应用”，系统再把这段需求转成标准化 JSON，再驱动后续的应用创建与配置流程。

把工程流水线变成可演示资产

最终不只留下一个应用，还会留下完整的视频、GIF、任务文件、执行报告和技术日志，方便对外展示与对内排查。

把复盘和复跑成本降下来

每个阶段的中间结果都落成 JSON。这样一旦某步失败，不需要从头猜，只要对着产物和日志定位问题即可。

Core Capabilities

主要功能

HAP Auto 的价值，在于它把多个原本分散的自动化脚本整合成一套可以解释、可以归档、可以拿去展示的系统。

需求驱动建应用

  • 终端和 Gemini 对话，收集应用目标、业务背景和工作表需求
  • 自动生成标准化需求 JSON，而不是把自然语言直接硬塞给执行脚本
  • 按需求驱动创建应用、工作表、图标和布局

结构配置自动化

  • 自动规划并创建工作表、关联字段、视图、表格筛选和导航样式
  • 结合 OpenAPI 和 Web 接口，补齐单一接口做不到的部分
  • 每一步都保留规划结果与执行结果，便于人工审阅

自动造数和业务展示

  • 按表结构生成模拟数据，支持关系一致性分析和修复
  • 让刚生成的应用不是空壳，而是一个能展示业务流程的 Demo
  • 更适合对外演示、客户沟通和内部汇报

浏览器录制与最终归档

  • 从自然语言 task 自动生成浏览器演示步骤
  • 录制出 MP4 和 GIF，直接变成演示资产
  • 自动汇总摘要页和技术日志，方便非技术人员和 AI 同时使用
Execution Story

项目整体执行过程

这个系统的关键，不在于某个单点脚本多复杂，而在于从输入、规划、执行到归档形成了一个完整闭环。

需求输入

用户用自然语言描述想做的应用，例如行业、场景、工作表和流程要求。

结构规划

Gemini 将需求转成结构化 spec，并继续规划工作表、字段、视图和布局方案。

自动执行

脚本调用 HAP OpenAPI 和页面接口，真正把应用创建出来，并写入配置和模拟数据。

录制成片

浏览器自动执行演示 task，录出 MP4 / GIF，并把业务摘要和技术日志一起归档。

Main Entry

run_app_to_video.py 的原理和流程

这是整个项目最适合对外展示的主入口。它把原本分散的多段流水线，收口为“一次对话，一次构建，一次录制”。

主流程命令

cd /Users/andy/Desktop/hap_auto
python3 scripts/run_app_to_video.py

运行后，用户只需要在第一步描述想要的应用。后续的应用构建、task 填充、录制和归档都会自动继续。

主流程的 3 个关键阶段

Stage 1
需求采集与应用创建

调用 agent_collect_requirements.py，由 Gemini 收集需求并自动执行建应用流水线。

Stage 2
生成演示任务

调用 fill_task_placeholders.py，把 task 模板里的应用、工作表、视图和 worksheetId 占位符填成真实值。

Stage 3
浏览器自动录制

调用 record/run_agent.py，在真实页面执行 task，并产出 MP4、GIF 和录制日志。

主入口背后的编排思想

A
不是直接“边想边做”，而是先生成 spec

先把需求沉淀成 JSON，再驱动执行器。这使得流程更稳定，也让后续每一步都可复盘。

B
不是只产出结果，还要产出证据链

主流程除了创建应用，还会自动复制关键 JSON、生成摘要和技术日志，避免“结果有了，但不知道怎么来的”。

C
不是录屏工具，而是演示资产工厂

它录的不是随机操作，而是从真实应用结构反推出来的浏览器演示步骤，所以视频更贴近业务表达。

Why It Matters

这个项目对外界的价值是什么

如果只说“它能自动化”，其实说得太轻了。HAP Auto 的真正价值，是把业务想法转成一个可运行、可展示、可复盘的数字资产包。

对客户

客户不用等长周期手工搭建，就能更快看到一个“有结构、有数据、有操作视频”的应用原型。

对团队

团队可以把重复性的应用搭建和演示准备工作交给系统，减少大量低价值的人肉配置。

对 AI 协作

每次运行都保留技术日志和关键 JSON，意味着 AI 不只是帮忙执行，还能真正参与排障、复盘和持续优化。

一句话总结

HAP Auto 不是一个“帮你点几下页面”的脚本集合，而是一套把需求、应用、数据和演示串成业务闭环的自动化系统。 它让外界看到的，不只是代码能力，而是“从业务理解到成果交付”的完整执行能力。
