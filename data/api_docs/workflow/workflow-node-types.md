## flowNode/add 接口说明

添加工作流节点时调用。

### Request Body

```json
{
  "processId": "69c230004aa00636bbbfa2c4",
  "prveId": "69c230004aa00636bbbfa2c5",
  "name": "节点名称",
  "typeId": 6,
  "actionId": "1",
  "appType": 1
}
```

| 参数 | 说明 |
|------|------|
| processId | 工作流进程 ID |
| prveId | 前一个节点 ID（新节点插在它后面）|
| name | 节点显示名称 |
| typeId | 节点类型（见下表）|
| actionId | 动作 ID（同 typeId 下区分具体动作，可不传）|
| appType | 应用类型（区分操作对象，可不传）|

### Response

```json
{
  "status": 1,
  "data": {
    "addFlowNodes": [{
      "id": "节点ID",
      "typeId": 6,
      "name": "新增记录",
      "actionId": "1",
      "appType": 1,
      "prveId": "...",
      "nextId": "...",
      "flowIds": [],
      "isException": true
    }],
    "updateFlowNodes": [...]
  }
}
```

## 关键注意事项

1. **typeId=6 + 不同 actionId + 不同 appType** = 不同的数据操作节点（新增/更新/查找/删除/校准）
2. **typeId=9** = 运算节点（数值/日期/函数/汇总），actionId 区分子类型
3. **typeId=29** = 循环节点，actionId 区分循环方式
4. **typeId=12** = 延时节点，actionId "301"=延时一段时间, "302"=延时到指定日期
5. **typeId=31** = AI 文本/数据对象生成，actionId 区分
6. **typeId=33** = AI Agent
7. `prveId` 决定节点插入位置，总是填当前最后一个节点的 ID
