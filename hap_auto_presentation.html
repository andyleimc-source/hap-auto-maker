强大的核心功能矩阵

我们不仅仅是调用 API，我们重塑了应用搭建的工作流。

💡 对话式需求驱动构建

通过终端与 Gemini 进行多轮智能对话，深度挖掘应用目标、业务背景和工作表需求。系统自动将自然语言结构化为符合标准的 JSON 规格书，彻底摆脱人工编撰代码和复杂配置的繁琐过程。

🏗️ 深度架构与应用组装

自动化覆盖绝大部分搭建工作：

  • 自动建表与配置字段：从文本生文本控件到复杂的关联关系（1-1/1-N/N-N）。
  • 多维视图模型：自动搭建看板、表格、画廊视图。
  • 定制化筛选器：根据业务逻辑生成多重表单过滤器。

📊 智能沙箱与数据造影

空荡荡的系统无法演示。引擎能依据设定的表结构，智能推断并自动生成极为逼真的业务数据，包含了复杂的关联关系一致性回填等，让刚生成的应用立即“活”起来。

🎥 无头浏览器演示录制

不再人工录像。内置 Playwright/browser-use 引擎驱动操作：

  • 描述驱动 (NLP to Actions)：由大白话指令控制录制行为，例如“点击左侧导航栏”。
  • 动态变量注入：自动读取上一环节生成的真实应用、视图、记录 ID 占位符。
  • 稳定输出：内置防掉帧与可见等待，自动输出 MP4 / GIF 资产。

化繁为简的执行过程

透明、可控的四步走策略，将抽象的想法映射为具体的数字世界资产。

01. 需求解析阶段

用户描述应用场景及业务要求。Gemini 介入处理，产出确切的系统结构规格书（Requirement Spec）。

02. 结构规划体系

引擎接管生成的 Spec，将其进一步细化为表设计规划、布局规划、视图规划等系列子方案。

03. 平台构建与造数

真实触发 API 初始化应用实例。完成元数据创建后，执行数据批量模拟和关联字段自动维护，赋予应用灵魂。

04. 演示录制与沉淀

根据应用特性动态生成录制脚本指令。环境接管浏览器自动展示，最终生成视频、日记、摘要，输出到标准归档目录。

拆解超级中枢：run_app_to_video.py

整个系统最高效的对外发声管道。它将繁复的生命周期收口为一次极简执行。

一键化奇迹

原本碎片化的“建应用”、“生成假数据”、“写演示剧本”、“录屏”流程，被 run_app_to_video.py 完全串联。只需启动脚本，剩下的交给机器调度。

核心设计哲学：
A. 坚持规格优先（先产出JSON规划再做动作）。
B. 保留强力证据链（生成的不仅是结果，更是可追溯的过程物）。
C. 资产导向（录制的不是机器乱点，而是模拟真实业务流向的演示视频）。

智能采集与应用实例化
