# 视图类型扩展与 AI 适用性判断设计

**日期**: 2026-04-07  
**状态**: 待实现

---

## 背景

当前生成的应用视图几乎全是列表、看板、画廊三种。问题根因：
1. `build_structure_prompt()` 的 viewType 列表只涵盖 0-5，资源(7)、地图(8) 从未出现在 AI 视野中
2. 日历(4)/甘特(5) 的触发条件仅靠工作表名关键词，太窄
3. 看板的 `KANBAN_FIELD_KEYWORDS` 包含「优先级/紧急程度/风险等级」这类分级字段（不流转），导致不适合的表也生成看板
4. AI prompt 对看板/日历/甘特缺乏正反例说明，AI 自行发挥

---

## 设计方案（方案 B：规则预筛 + AI 最终决策）

### 数据流

```
字段列表
  ↓
classify_fields()          # 不改
  ↓
suggest_views()            # 修改：输出「候选视图」(字段层过滤)
  ↓
build_structure_prompt()   # 修改：候选列表注入 prompt，AI 做语义判断
  ↓
AI 输出视图规划 JSON
  ↓
validate_structure_plan()  # 修改：新增字段存在性校验
```

**不改的文件**：`view_types.py`、`constraints.py`、`create_views_from_plan.py`

## 修改 1：`scripts/hap/planning/view_planner.py`

### 1a. 收窄看板关键词

```python
# 只保留真正的阶段流转语义
KANBAN_FLOW = {
    "状态", "阶段", "进度", "步骤", "环节",
    "审批", "审核", "审查", "审定",
}
# 删除：优先级/紧急程度/严重程度/风险等级 → 分级字段，不流转，不适合看板
```

### 1b. 修改 `suggest_views()` — 各视图候选规则

| 视图 | 前置条件（规则层） | 是否交 AI 判断 |
|------|------|------|
| 看板(1) | type=9/11 字段名含 KANBAN_FLOW | 是：AI 确认字段是否真的代表阶段流转 |
| 日历(4) | 有 ≥1 个日期字段 | 是：AI 判断数据是否有时间轴意义 |
| 甘特(5) | 有 ≥2 个日期字段 | 是：AI 判断是否有起止时间跨度业务意义 |
| 资源(7) | 有 type=26 成员字段 + ≥2 个日期字段 | 是：AI 判断是否有人员时间分配场景 |
| 地图(8) | 有 type=40 定位字段 | 否：有字段直接推荐 |
| 画廊(3) | 有 type=14 附件字段 | 否：有字段直接推荐 |

`suggest_views()` 输出新增 `candidate: True` 标记，区分「AI 待判断候选」和「确定推荐」：

```python
# 候选视图（交 AI 判断）
{
    "viewType": 4,
    "name": "日历视图",
    "candidate": True,
    "eligible_reason": "有日期字段「截止日期」(type=15)",
    "calendarcid": "<date_field_id>",
}

# 确定推荐（无需 AI 判断）
{
    "viewType": 8,
    "name": "地图视图",
    "candidate": False,
    "eligible_reason": "有定位字段「位置」(type=40)",
    "latlng": "<location_field_id>",
}
```

### 1c. 新增 `validate_structure_plan()` 校验

AI 输出 viewType=7 时，回查该工作表是否有成员字段 + 两个日期字段。  
不符合则从结果中剔除并打印警告（不中断流程）。日历(4)/甘特(5) 同理。

## 修改 2：`build_structure_prompt()` 中的 AI 判断框架

### 2a. 扩展视图类型列表（新增 6/7/8）

在 prompt 的 viewType 列表中补充：
- `7. 资源视图` — 按成员/字段分组的时间线，需要成员字段 + 开始/结束日期
- `8. 地图视图` — 需要 type=40 定位字段

### 2b. 候选视图注入

```
## 候选视图（由字段分析预生成，请判断是否采纳）

工作表「任务管理」:
  - 候选: viewType=4 日历视图（有字段「截止日期」type=15）
  - 候选: viewType=5 甘特图（有「开始日期」+「截止日期」）
  - 候选: viewType=7 资源视图（有成员字段「负责人」+ 两个日期字段）
```

### 2c. 判断标准段落（含正反例）

```
## 视图采纳判断标准

**看板(1)**：仅当字段是「阶段流转」时采纳。
  ✅ 适合：审批状态（待审/审批中/已通过）、项目阶段（规划/开发/测试/上线）
  ❌ 不适合：优先级（高/中/低）、类型（合同类/服务类）、来源（官网/转介绍）
  ⚠️ 优先级/等级/类型/分类字段一律不用看板，即使是单选字段。

**日历(4)**：仅当数据有「时间维度上的分布」意义时采纳。
  ✅ 适合：会议记录、活动排期、任务截止日、排班记录
  ❌ 不适合：员工档案（有入职日期但不是时间轴数据）、产品列表（有上架日期但不按日查看）

**甘特图(5)**：仅当数据有「起止时间跨度」业务意义时采纳。
  ✅ 适合：项目/任务/计划/排产/版本发布
  ❌ 不适合：合同（有签约日+到期日，但不是项目执行时间线）

**资源视图(7)**：仅当需要「查看某人在某时间段的工作安排」时采纳。
  ✅ 适合：排班表、任务分配、项目资源排期
  ❌ 不适合：普通任务表（即使有负责人和日期，但不需要按人查看时间线）
```
