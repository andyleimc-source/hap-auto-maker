## 一、工作流基本概念

**工作流 = 1个触发器 + N个动作节点**

- **触发器**：流程启动的开关，满足条件才启动
- **动作节点**：自动执行的任务（数据操作、通知、审批等）

**工作流能解决的问题：**
1. 数据自动化处理（数据变更时新增/修改其他表数据）
2. 业务流程审批控制（销售订单、请假等审批）
3. 通知内外部人员（站内消息、邮件、短信）
4. 外部系统数据对接（推送/获取外部系统数据）

## 二、9种触发方式

| 触发类型 | 说明 | 关键参数 |
|---------|------|---------|
| **工作表事件触发** | 新增/更新/删除记录时触发 | 监听表、触发方式、触发条件 |
| **定时触发** | 设定循环周期时间触发 | 周期配置 |
| **按日期字段触发** | 到达表中某日期字段值时触发 | 日期字段 |
| **组织人员事件触发** | 有人加入/离职时触发 | 事件类型 |
| **外部用户事件触发** | 外部用户注册/登录/注销/停用时触发 | 事件类型 |
| **Webhook触发** | 外部系统推送数据到指定URL触发 | Webhook URL、数据样本 |
| **封装业务流程(PBP)** | 被其他流程/自定义页面按钮调用触发 | - |
| **子流程** | 不能直接创建，通过父流程的子流程节点触发 | - |
| **自定义按钮触发** | 点击工作表中的自定义动作按钮触发 | 按钮ID |

### 工作表事件触发 - 4种触发方式
- **新增或更新记录时**：可指定触发字段（减少不必要触发）
- **仅新增记录时**：有新记录时触发
- **仅更新记录时**：已有记录内容被修改时触发，支持指定触发字段
- **删除记录时**：删除记录后触发

### Webhook触发要点
- 每个Webhook流程自动生成唯一URL
- 数据解析方式：从请求范例生成 / 从JSON数据范例生成 / 从key-value pairs生成
- 支持设置必填参数（缺少时返回错误）
- 工作流发出请求的IP：`123.59.59.220` 和 `106.75.14.110`（随机一个）

## 三、节点类型汇总（约40种）

### 数据操作节点
| 节点 | 功能 | 关键说明 |
|------|------|---------|
| **新增记录** | 向指定工作表新增记录 | 结合"获取多条数据"可批量新增 |
| **更新记录** | 更新记录字段值 | 可修改/清空/增减值；需先有节点指向记录 |
| **删除记录** | 删除记录 | 支持彻底删除（不进回收站） |
| **获取单条数据** | 查询并获取1条记录 | 6种获取方式（见下） |
| **获取多条数据** | 查询并获取多条记录 | 8种获取方式（见下） |

**获取单条数据的6种方式：**
1. 查询并更新记录（查询+更新一步到位，支持返回更新前数据）
2. 查询并删除记录
3. 查询工作表（仅查询）
4. 从多条数据中获取（从"获取多条数据"节点的结果中取1条）
5. 获取关联记录（从关联表/子表中获取）
6. 从记录链接获取（解析文本字段中的分享链接）

**获取多条数据的8种方式：**
1. 查询并批量更新（最多1000行）
2. 查询并批量删除（最多1000行）
3. 查询工作表（最多100行供后续节点；传子流程最多10000行）
4. 查询聚合表
5. 获取关联记录（最多1000条）
6. 获取批量新增结果
7. 获取操作明细（审批/填写节点的操作数据）
8. 获取数组对象（接收代码块/API/PBP输出的对象数组）

### 逻辑控制节点
| 节点 | 功能 |
|------|------|
| **分支节点** | 按条件走不同分支；4种结构：并行/唯一/审批结果/获取数据结果 |
| **子流程** | 调用子流程，可传递多条数据逐条处理 |
| **循环** | 循环执行节点 |
| **延时节点** | 延时一段时间后继续执行 |
| **中止流程** | 主动中止当前流程 |

**分支节点4种结构：**
- **并行分支**：所有满足条件的分支依次执行（左到右）
- **唯一分支**：只执行第一个满足条件的分支（if-else逻辑）
- **审批结果分支**：固化为"通过"和"不通过"两个分支
- **获取数据结果分支**：固化为"有数据"和"无数据"两个分支

### 通知节点
| 节点 | 功能 |
|------|------|
| **抄送节点** | 发站内消息+记录链接，进入流程待办，需手动查看才消减待办数 |
| **发送站内通知** | 仅发消息内容，不进待办，查看后立即消减 |
| **发送邮件** | 向指定用户发邮件 |
| **发送短信** | 0.05元/条 |
| **发送服务号消息** | 微信服务号消息 |
| **界面推送节点** | 推送到界面 |
| **声音播报** | 声音通知 |

### 审批节点
| 节点 | 功能 |
|------|------|
| **发起审批流程** | 推荐使用；可在工作表直接审批，支持管理员介入/中止/替换操作人 |
| **审批节点** | 简单审批，功能少于"发起审批流程" |
| **填写节点** | 指定人员填写数据后继续流程 |

### 外部集成节点
| 节点 | 功能 |
|------|------|
| **发送API请求** | GET/POST/PUT/DELETE/HEAD/PATCH，支持推送和获取数据 |
| **调用已集成API** | 调用平台已配置的第三方API |
| **代码块节点** | 执行JS或Python代码 |
| **JSON解析节点** | 通过JSONPath解析代码块/API返回的JSON |
| **获取链接** | 获取记录分享链接 |
| **获取页面快照** | 获取页面截图 |
| **获得记录打印文件** | 生成打印文件 |
| **获取支付链接** | 生成支付链接（需工作表开启支付） |

### AI节点
| 节点 | 功能 |
|------|------|
| **AI生成文本** | AI生成文字内容 |
| **AI生成数据对象** | AI生成结构化数据 |

### 人员数据节点
| 节点 | 功能 |
|------|------|
| **获取单条人员数据** | 获取部门/组织角色中的单个人员 |
| **获取多条人员数据** | 批量获取人员数据 |
| **获取单条外部人员数据** | 获取外部用户数据 |
| **邀请外部用户** | 发送外部用户邀请 |
| **更新外部用户信息** | 更新外部用户字段 |

### 其他节点
| 节点 | 功能 |
|------|------|
| **运算节点** | 数值/日期运算，日期加减时间等 |
| **校准** | 数据校准 |
| **创建任务** | 创建系统任务 |

## 四、全局配置

### 运行方式（仅适用于：仅新增/删除/自定义动作触发）
- **并行执行**：同时执行，速度快，适合互不影响的场景
- **顺序执行**：逐条按序执行，遇到子流程/审批节点不等待完成
- **严格串行**：完全执行完上一条再执行下一条（包括子流程）

### 其他全局配置
- **流程错误通知人**：流程出错时通知指定人
- **节点日志**：开启后可在历史中看每个节点执行时的实际数据（保留90天）
- **触发者查看**：是否在触发者的"我发起的"待办中显示
- **触发其他工作流**：本流程的数据操作是否允许触发其他工作流
- **数据格式**：数值小数位数处理方式、日期精度（精确到分钟/秒）

## 五、触发信息

### 数据源ID 结构
格式：`触发流程的记录ID|触发者的用户ID|触发时间戳`

特殊触发者ID：
- API触发：`user-api`
- 工作流触发：`user-workflow`
- 公开表单提交：`user-publicform`

### 执行实例ID
格式类似：`62c7ff5837a28b63af673b05`

查看实例详情地址：`/2/实例ID`（在工作流配置页URL后拼接）

## 六、管理操作

### 单个工作流管理
- 修改名称/说明
- 调整全局配置
- 测试流程（支持：工作表事件/定时/按日期字段/按钮/子流程/PBP；不支持：人员事件/循环/Webhook）
- 发布/更新发布
- 转移到其他应用（仅Webhook/定时/人员事件/PBP类型）
- 转为子流程或PBP
- 删除（进回收站，60天后彻底删除；付费版可恢复）

### 历史版本与执行详情
- 查看流程实例列表和每个实例的节点执行状态
- 手动中止流程实例（批量支持）
- 重新执行失败/中止的流程（从失败节点开始，不用重头触发）
- 查看历史发布版本、版本回滚（需手动"更新发布"才生效）

### 组织级工作流管理
- 查看所有工作流列表、当月执行数量
- 工作流监控：实时排队数、5分钟新增/消费数
- 排队预警设置
- 暂停/恢复工作流执行
- 重置排队计数

## 七、代码块节点要点

### 运行环境
- **Node.js v10.16.3**：已安装 moment、lodash、node-fetch、form-data、xml2js、node-rsa、uuid、pinyin、sm-crypto
- **Python v3.7.5**：已安装 requests、xmltodict、pycryptodome、dingtalk-sdk、beautifulsoup4、rsa、uuid、xpinyin、gmssl、cryptography、lxml、pandas、numpy、matplotlib

### 代码规范
```javascript
// 接收参数
var value = input.参数名;

// 输出结果
output = { 字段名: 值 };
```
- 执行时间不能超过10秒
- 接收"获取多条数据"时最多100行

## 八、发送API请求节点要点

- 支持：GET/POST/PUT/DELETE/HEAD/PATCH
- 请求时长不能超过10秒
- 工作流出口IP：`123.59.59.220` 和 `106.75.14.110`（随机）
- 支持 Header 配置（授权Token等）
- 支持网络代理（需管理员在组织后台配置）
- 支持设置请求失败时的错误状态码和消息

## 十三、常见开发注意事项

1. **触发循环问题**：流程A的更新节点修改了表1，如果流程B监听表1，B的触发器必须指定了触发字段才能被A触发（防止死循环）
2. **记录触发锁**：记录A已经在跑流程B时，再次修改记录A不会重新触发流程B
3. **本月执行数用完**：工作流会停止执行，需购买升级包
4. **测试消息发送**：测试编辑中的流程时，通知/抄送等消息只发给测试者本人，不发给节点配置的人
5. **按日期字段触发**：注意精确到分钟还是秒的全局配置
6. **子流程数据量**：最多传10000行，不要超限
7. **API速率限制**：同一IP的QPS：新增/编辑/删除=20次/秒，查询=50次/秒；超出返回 `error_code:90001`
