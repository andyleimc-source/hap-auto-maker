更新时间：2026-03-13  
用途：为 `scripts/hap/*workflow*.py` 提供稳定、可落盘、可复用的工作流知识摘要。  
说明：这里只保留脚本设计和排障直接需要的事实，不做整站镜像。

## 1. 创建流程总览
- 工作流的核心链路是：选择触发方式 -> 配置流程节点 -> 保存草稿 -> 发布 -> 启停。
- 草稿状态下可以反复调整节点、字段映射、流程参数。
- 修改已上线工作流后，需要重新发布，旧版本不会自动替换为最新配置。
- 测试时通常依赖具体记录或具体触发上下文，因此脚本联调要准备测试数据。

## 2. 工作表事件触发
- 工作表事件触发适合监听单张工作表上的数据变化。
- 典型触发模式包括：新增记录、更新记录、删除记录、创建或更新。
- 更新触发可进一步限定“指定触发字段”，只有这些字段变化时才触发。
- 工作表触发通常还会配合触发条件，避免所有变更都启动工作流。
- 对脚本而言，最小必要上下文是：触发工作表、触发事件、指定字段列表、条件列表。

## 3. 定时触发
- 定时触发用于按固定时间规则启动工作流，而不是等待单条记录变化。
- 关键参数是：开始时间、结束时间、循环周期、具体执行时间。
- 周期可以是每天、工作日、指定间隔等形式；一期脚本先统一收敛到 `daily` 或 `weekday` 两类。
- 定时触发常见用途是：定期提醒、定期汇总、定期创建记录、定期更新状态。

## 4. 节点与运行配置
- 流程由触发器和后续节点组成。
- 常见节点类型覆盖通知、数据更新、创建记录、延时、审批、Webhook、代码块等。
- 一期 Gemini 规划先收敛为三种最容易结构化的动作节点：
  - `update_fields`
  - `create_record`
  - `send_notice`
- 这样做的目的是先把 JSON 契约和自动化链路稳定下来，后续再扩展节点白名单。

## 5. 发布、测试、启停
- 工作流创建后通常需要发布，发布成功后才进入可运行状态。
- 如果是已启用工作流，修改配置后需要重新发布才能让新版本生效。
- 启停是独立动作，很多平台场景里“已发布”不等于“已启用”。
- 一期创建脚本按三段式设计：
  1. 保存/创建草稿
  2. 发布
  3. 启用

## 6. 触发信息
- 触发信息通常包含触发记录、触发时间、触发人以及流程实例上下文。
- 对脚本和 AI 规划来说，这意味着：
  - 新增/更新触发适合围绕当前记录做字段更新或通知
  - 定时触发更适合做批量扫描、周期创建或周期提醒
- 触发信息是后续补充更复杂节点映射时的重要依据。

## 7. 一期脚本落地约束
- 规划层固定输出 3 个工作流：
  - `worksheet_create_trigger`
  - `worksheet_update_trigger`
  - `scheduled_trigger`
- schema 导出层只保留工作流最小必要元数据：应用、工作表、字段的名称、ID、类型。
- 创建层先支持 dry-run 和适配器骨架；真实私有接口细节以后续 HAR / 抓包为准补齐。
